매서드 시그니처를 신중히 설계하라.

1. 메서드이름은 신중히 짓자. => item68을 따르자. 기본적으로 통용되는 명령법을 따르는게 제일 좋다.

2. 편의 메서드를 너무 많이 만들지말자.
메서드가 많으면 당연히 익히고, 사용하고, 문서화하고 테스트하고, 유지보수하기 어렵다.
인터페이스도마찬가지이다.

3.매개변수는 4개이하가 좋다.
매개변수가 4개가 넘어가면 매개변수를 전부 기억하기가 쉽지않다.
특히,같은 타입의 매개변수가 여러개가 연달아 나오는 경우가 특히 해롭다.

    매개변수양을 줄이기위한 스킬들

    3-1, 여러메서드로 쪼갠다.
    3-2, 매개변수를 여러개로 묶어주는 도우미 클래스를 만드는 것이다.
    3-3, 1과2를 혼합한것.
